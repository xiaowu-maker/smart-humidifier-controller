# 05 - GPIO 分配说明

> **项目**: SmartHumidifier — ESP32 DevKit V1 智能湿度控制器  
> **文档版本**: 1.0  
> **最后更新**: 2026-07-11

---

## 1. 概述

本文档详细说明 SmartHumidifier 项目中 ESP32 DevKit V1 开发板的 GPIO（通用输入输出）引脚分配方案。合理的 GPIO 分配是硬件稳定性和软件可维护性的基础，在分配过程中需要兼顾 ESP32 芯片的硬件约束、外设电气特性、PCB 布线便利性以及软件可读性。

本方案涉及的板载外设如下：

| 编号 | 外设 | 接口类型 | 所需引脚数 |
|------|------|----------|-----------|
| 1 | AHT20 温湿度传感器 | I2C | 2（SDA / SCL） |
| 2 | 0.96寸 OLED SSD1306 | I2C | 2（SDA / SCL，与 AHT20 共享总线） |
| 3 | 单路继电器模块 | GPIO 高低电平 | 1 |
| 4 | 三色 LED RGB 共阴极 | GPIO 高低电平 | 3（红 / 绿 / 蓝） |
| 5 | 有源蜂鸣器 | GPIO 高低电平 | 1 |
| 6 | 按钮 A（模式切换） | GPIO 输入（上拉） | 1 |
| 7 | 按钮 B（恢复默认） | GPIO 输入（上拉） | 1 |

**合计占用物理引脚数**: 11 个（I2C 总线共享 2 个，其余各占 1 个）。

> 注意：ESP32 DevKit V1 共引出 30 个 GPIO（部分型号为 38 引脚版本，本文以最通用的 30 引脚版本为例），除去供电、地线、USB 串口等专用引脚后，可用 GPIO 资源仍十分充裕。

## 2. ESP32 引脚特殊约束

在分配 GPIO 之前，必须充分了解 ESP32 芯片的以下关键限制：

### 2.1 Strapping 引脚（启动配置引脚）

ESP32 在上电或复位时会采样一组称为 **Strapping 引脚** 的电平状态，用于决定芯片的启动模式、电压、闪存配置等。如果在这些引脚上连接了外部电路（尤其是下拉电阻或按键），可能导致芯片无法正常启动或进入下载模式。

| GPIO | 功能 | 默认状态 | 注意事项 |
|------|------|---------|---------|
| **GPIO0** | 启动模式选择 | 上拉（高电平） | 拉低进入下载模式（Flash Boot）。**不应**在外部接下拉电路，否则上电即进入下载模式。 |
| **GPIO2** | 启动模式选择 | 下拉（低电平） | 拉高且 GPIO0 为低时进入下载模式。启动过程中必须保持高阻或低电平。 |
| **GPIO5** | VDD_SDIO 电压选择 | 上拉（高电平） | 拉低会将 SDIO 供电设为 1.8V（多数模组需要 3.3V）。**必须避免**在启动时被外部拉低。 |
| **GPIO12** | VDD_SDIO 电压选择（MTDI） | 下拉（低电平） | 拉高会将 SDIO 供电设为 1.8V。内部有下拉电阻，但仍需避免外部上拉。 |
| **GPIO15** | 启动日志输出（MTDO） | 上拉（高电平） | 拉低会禁止 U0TXD 输出启动日志。**不应**在外部拉低。 |

**本项目的分配原则**: Strapping 引脚（GPIO0、GPIO2、GPIO5、GPIO12、GPIO15）一律避免作为输出型外设（继电器、LED、蜂鸣器）的驱动引脚，也不连接按钮。其中 GPIO0 和 GPIO2 在某些设计中可用于按钮（但须谨慎处理上拉/下拉方向），本方案不采用以简化设计。

### 2.2 仅输入引脚

**GPIO34 ~ GPIO39** 为仅输入引脚，**没有内部上拉/下拉电阻**，也不能配置为输出模式。这些引脚适合作为只读的数字输入（如按钮），但需外部上拉。

本方案中这些引脚留作备用或 AD 采集扩展，不纳入核心分配。

### 2.3 ADC2 与 Wi-Fi 冲突

ESP32 的 ADC2 模块（GPIO25、GPIO26、GPIO27、GPIO14、GPIO12、GPIO13、GPIO4、GPIO2）在 Wi-Fi 开启时无法使用。如果项目启用 Wi-Fi（例如用于远程监控或 OTA），则 ADC2 引脚不能可靠地进行模拟采集。

本项目当前无模拟采集需求，且不严重依赖 ADC，故不构成阻塞。

### 2.4 启动日志串口

**U0TXD (GPIO1)** 和 **U0RXD (GPIO3)** 默认被 USB 转串口芯片占用，用于程序烧录和串口监视器输出。这两个引脚不应被外设占用，否则会影响调试和烧录。

**U1TXD (GPIO10)** 和 **U1RXD (GPIO9)** 在某些 ESP32 模组中也被引出，可复用为普通 GPIO，但需注意电气兼容性。

### 2.5 JTAG / 调试引脚

GPIO12 (MTDI)、GPIO13 (MTCK)、GPIO14 (MTMS)、GPIO15 (MTDO) 在启用 JTAG 调试时被占用。本项目不使用 JTAG，因此这些引脚可作普通 GPIO 使用，但需避开 Strapping 风险。

## 4. 分配方案详细说明

### 4.1 I2C 总线：GPIO21 (SDA) + GPIO22 (SCL)

- **依据**: ESP32 的 I2C 硬件控制器默认映射到 GPIO21 (SDA) 和 GPIO22 (SCL)。虽然 ESP32 的 I2C 可以通过 GPIO MUX 任意映射到几乎所有引脚，但使用默认引脚可以获得最佳的时序性能和最简洁的代码配置。
- **总线共享**: AHT20（地址 0x38）与 SSD1306 OLED（地址 0x3C）具有不同的 I2C 设备地址，可以并联在同一总线上，仅需两个上拉电阻（通常 4.7kΩ ~ 10kΩ 至 3.3V）即可正常工作。
- **抗干扰**: 使用硬件 I2C 控制器而非软件模拟 I2C（bit-banging），避免 CPU 被频繁的中断占用，同时获得更稳定的时钟信号。
- **Arduino 默认**: 在 Arduino-ESP32 框架中，`Wire.begin()` 默认使用 GPIO21/GPIO22。

### 4.2 继电器控制：GPIO4

- 选择 GPIO4 的原因：
  - 远离 Strapping 引脚（GPIO0、GPIO2、GPIO5、GPIO12、GPIO15），启动时序不受影响。
  - GPIO4 不属于 ADC2（Wi-Fi 冲突组），如需启用 Wi-Fi 不会造成影响。
  - 继电器模块通常包含光耦隔离和续流二极管，对 GPIO 的干扰较小。
- **驱动逻辑**: 建议采用 **低电平有效** 设计（GPIO 输出低 → 继电器吸合；输出高 → 继电器释放）。原因：ESP32 上电复位瞬间 GPIO 默认状态为高电平（输入模式 + 内部上拉），若设计为高电平有效会导致上电瞬间继电器误动作。低电平有效则在上电时继电器默认断开，待系统初始化完成后由软件主动控制。
- **保护建议**: 在 GPIO4 与继电器 IN 引脚之间串接 1kΩ 限流电阻，继电器模块端已有光耦隔离时可省略。

### 4.3 RGB LED：GPIO16 (R) / GPIO17 (G) / GPIO18 (B)

- 选择 GPIO16/17/18 的原因：
  - 三者连续排列（16 → 17 → 18），在 PCB 布局时走线集中、便于区分。
  - 全部避开 Strapping 引脚。
  - 不属于 ADC2 冲突域。
  - 均为标准 3.3V 逻辑输出，驱动共阴极 RGB LED 时经 220Ω 限流电阻后直接连接 LED 阳极，阴极统一接地。
- **驱动逻辑**: 高电平点亮。PWM 功能可复用到这些引脚（ESP32 大部分 GPIO 均可输出 PWM）以实现呼吸灯或颜色渐变效果。
- **注意事项**: 红色 LED 正向压降较低（约 1.8V ~ 2.0V），绿色和蓝色较高（约 2.8V ~ 3.2V），建议不同通道使用不同限流电阻值以使亮度均衡：
  - R 通道: 330Ω
  - G 通道: 220Ω
  - B 通道: 220Ω
  或在软件中通过 PWM 占空比校准。

### 4.4 有源蜂鸣器：GPIO19

- 选择 GPIO19 的原因：
  - 远离敏感信号（I2C、按钮等），避免电磁干扰相互耦合。
  - 不属于 ADC2 冲突域。
  - GPIO19 与 I2C 的 SCL (GPIO22) 物理位置较近，蜂鸣器工作时产生的电磁干扰可能影响 SCL 信号。因此建议在蜂鸣器两端并联续流二极管（SS14 肖特基）或 RC 吸收电路以抑制反向电动势。
- **驱动逻辑**: 高电平发声。有源蜂鸣器内部已集成振荡源，只需 GPIO 输出高电平即可发出固定频率声音。PWM 驱动可用于调节音量（通过改变占空比）。
- **保护建议**: 若蜂鸣器额定电流较大（>30mA），需通过 NPN 三极管（如 S8050）或 MOS 管驱动，不可由 GPIO 直接驱动。

### 4.5 按钮 A（模式切换）：GPIO13 带内部上拉

- 选择 GPIO13 的原因：
  - 属于 MTDI/MTCK/MTMS 组中的 MTCK，但本项目未使用 JTAG，故可作普通 GPIO。
  - GPIO13 不是 Strapping 引脚，启动电平不影响模式选择。
  - 物理位置靠近 GPIO14（按钮 B），便于 PCB 布局中成对放置。
- **电气设计**: 启用 ESP32 内部上拉电阻（典型值约 45kΩ），按钮另一端接地。常态下 GPIO13 为高电平；按下时被拉低，产生下降沿。
- **软件读取**: 使用 `digitalRead()` 配合去抖（debounce）算法（延迟 20~50ms 后再次确认电平），或使用硬件 GPIO 中断（GPIO_INTR_NEGEDGE）触发回调。

### 4.6 按钮 B（恢复默认）：GPIO14 带内部上拉

- 选择 GPIO14 的原因：
  - GPIO14 为 JTAG 信号 MTMS，在本项目中不作 JTAG 使用。
  - GPIO14 不属于 Strapping 引脚，启动阶段电平不被采样。
  - 与 GPIO13 并列，构成按钮组。
- **电气设计**: 与按钮 A 相同，启用内部上拉，按下拉低。
- **安全设计**: 按钮 B 定义为"恢复出厂默认设置"，为防止误触，建议在软件层面实现长按确认机制（如连续按下超过 3 秒才执行恢复操作），或通过组合键（同时按下 A + B）。

## 5. 需避开的引脚详解

### 5.1 绝对避开的引脚

| GPIO | 原因 | 说明 |
|------|------|------|
| **GPIO0** | Strapping: 启动模式 | 拉低进入下载模式。若外部接下拉或按钮，上电即进入烧录模式导致无法正常启动。 |
| **GPIO1** | U0TXD | USB 串口发送引脚，被 CP2102/CH340C 占用。 |
| **GPIO3** | U0RXD | USB 串口接收引脚，被 CP2102/CH340C 占用。 |
| **GPIO5** | Strapping: SDIO 电压 | 拉低使闪存工作于 1.8V（多数模组需要 3.3V），可能造成闪存读写异常。 |
| **GPIO12** | Strapping: SDIO 电压 | 拉高使闪存工作于 1.8V。内部有下拉电阻，仍应避免外部上拉。 |

### 5.2 建议谨慎使用的引脚

| GPIO | 原因 | 说明 |
|------|------|------|
| **GPIO2** | Strapping: 启动模式之一 | 启动时需保持低电平，不能接外部上拉；如果作为输出，需确保上电时为低。 |
| **GPIO15** | Strapping: 启动日志输出 | 拉低会关闭启动日志输出，调试不便。 |

### 5.3 引脚分配规避策略总结

在分配时遵循以下优先级：

1. **第一优先级**: 固定功能引脚（U0TXD/U0RXD、I2C 默认引脚）——不做它用。
2. **第二优先级**: Strapping 引脚（GPIO0、GPIO2、GPIO5、GPIO12、GPIO15）——不分配，留空或仅作谨慎输入。
3. **第三优先级**: 仅输入引脚（GPIO34~39）——留作备用或模拟输入扩展。
4. **第四优先级**: ADC2 引脚（GPIO25、26、27、14、12、13、4、2）——当前不涉及 ADC，但如未来启用 Wi-Fi 需注意 ADC2 不可用。
5. **可用引脚**: 剩余未占用的引脚（GPIO4、13、14、16、17、18、19、21、22、23、27、32、33）中，优先选择在物理位置上便于布线且相互干扰最小的组合。

## 6. 与 config.h 中的引脚定义对应关系

以下是 `config.h` 中的宏定义示例，与上述 GPIO 分配方案一一对应。此文件应被所有源文件引用，确保引脚定义集中管理、避免硬编码。

```c
#ifndef CONFIG_H
#define CONFIG_H

// ============================================================
// I2C 总线引脚定义
// ESP32 硬件 I2C 默认映射，使用 Wire.begin() 默认值
// ============================================================
#define I2C_SDA_PIN         21
#define I2C_SCL_PIN         22

// ============================================================
// I2C 设备地址
// ============================================================
#define AHT20_ADDR          0x38    // AHT20 默认 I2C 地址
#define OLED_ADDR           0x3C    // SSD1306 OLED 默认 I2C 地址

// ============================================================
// OLED 显示参数（SSD1306 128x64）
// ============================================================
#define OLED_WIDTH          128
#define OLED_HEIGHT         64
#define OLED_RESET_PIN      -1      // 未使用独立复位引脚

// ============================================================
// 继电器控制引脚 — 低电平有效
// ============================================================
#define RELAY_PIN           4

// ============================================================
// RGB LED 引脚定义（共阴极，高电平点亮）
// ============================================================
#define RGB_RED_PIN         16
#define RGB_GREEN_PIN       17
#define RGB_BLUE_PIN        18

// 可选的 PWM 通道分配（LEDC 共 16 通道，0~15）
#define RGB_RED_CHANNEL     0
#define RGB_GREEN_CHANNEL   1
#define RGB_BLUE_CHANNEL    2
#define RGB_PWM_FREQ        5000    // 5 kHz
#define RGB_PWM_RESOLUTION  8       // 8-bit: 0~255

// ============================================================
// 有源蜂鸣器引脚 — 高电平发声
// ============================================================
#define BUZZER_PIN          19

// ============================================================
// 按键引脚定义（内部上拉，按下为低）
// ============================================================
#define BTN_A_PIN           13      // 模式切换
#define BTN_B_PIN           14      // 恢复默认设置

// 按键防抖时间（毫秒）
#define BTN_DEBOUNCE_MS     50

// 按键 B 长按确认时间（毫秒）
#define BTN_B_LONG_PRESS_MS 3000

// ============================================================
// 系统运行参数
// ============================================================
#define SERIAL_BAUD_RATE    115200  // 串口监视器波特率
#define SENSOR_READ_INTERVAL_MS  2000  // 传感器读取周期（毫秒）
#define RELAY_UPDATE_INTERVAL_MS 1000  // 继电器状态刷新周期（毫秒）

#endif  // CONFIG_H
```

> **注意**: 以上宏定义仅为推荐配置。实际项目中可根据需求调整，但必须保持 GPIO 编号与硬件连接一致。

## 7. 未分配引脚及未来扩展建议

以下 ESP32 引脚在本方案中未分配，可供后续功能升级使用：

| GPIO | 可用作 | 注意事项 |
|------|--------|---------|
| GPIO23 | 通用 I/O、SPI MOSI、PWM | 完全自由，推荐用于新增外设 |
| GPIO27 | 通用 I/O、ADC2_CH7、PWM | Wi-Fi 启用时 ADC2 不可用；作为数字 I/O 无限制 |
| GPIO32 | 通用 I/O、ADC1_CH4、PWM | 推荐用于模拟传感器输入（如湿度电位计） |
| GPIO33 | 通用 I/O、ADC1_CH5、PWM | 推荐用于模拟传感器输入 |
| GPIO25 | 通用 I/O、ADC2_CH8、DAC1 | 可用于模拟输出 |
| GPIO26 | 通用 I/O、ADC2_CH9、DAC2 | 可用于模拟输出 |
| GPIO34~39 | 仅输入 | 不可输出，无内部上拉，适合按键或外部中断源 |

推荐的扩展方向：
- **DHT22 单总线传感器**: 可用 GPIO23 或 GPIO27 作为单总线数据引脚。
- **土壤湿度传感器（模拟）**: 可用 GPIO32 或 GPIO33（ADC1）进行模拟采集，ADC1 不受 Wi-Fi 影响。
- **ESP-NOW / 无线通信**: 无需额外引脚。
- **MicroSD 卡模块**: 需要使用 SPI（GPIO23=MOSI, GPIO19=MISO, GPIO18=SCK, GPIO5=CS）——注意这与本分配方案中 GPIO18/19 的现有用途冲突，如需 SD 卡应调整分配。

## 8. 调试与验证建议

### 8.1 上电验证清单

在首次焊接或搭建面包板后，请按以下步骤验证引脚分配的正确性：

1. **测量 Strapping 引脚电压**: 上电后用万用表测量 GPIO0、GPIO2、GPIO5、GPIO12、GPIO15 的对地电压，确认未被非预期拉低或拉高。
2. **I2C 总线扫描**: 上传 I2C 扫描程序到 ESP32，确保能够检测到地址 0x38（AHT20）和 0x3C（OLED）。
3. **按钮电平测试**: 在不按下按钮时读取 GPIO13、GPIO14 应为 HIGH，按下后为 LOW。
4. **逐通道测试输出**: 分别将 RELAY_PIN、RGB_R/G/B_PIN、BUZZER_PIN 置高和置低，观察对应外设响应。

### 8.2 代码示例：I2C 设备扫描

```cpp
#include <Wire.h>

void setup() {
  Serial.begin(115200);
  Wire.begin(I2C_SDA_PIN, I2C_SCL_PIN);  // 21, 22
  Serial.println("I2C Scanner");
  for (uint8_t addr = 1; addr < 127; addr++) {
    Wire.beginTransmission(addr);
    if (Wire.endTransmission() == 0) {
      Serial.printf("Found device at 0x%02X\n", addr);
    }
  }
}

void loop() {}
```

### 8.3 常见问题诊断

| 现象 | 可能原因 | 解决措施 |
|------|---------|---------|
| 上电后芯片反复重启 | GPIO0 或 GPIO2 被外部拉低 | 检查 GPIO0/GPIO2 外部电路，移除下拉电阻 |
| I2C 设备检测不到 | SDA/SCL 接线错误、缺少上拉电阻 | 确认接线，在 SDA/SCL 上各加 4.7kΩ 上拉至 3.3V |
| 按键无响应 | 内部上拉未启用，引脚浮空 | 确认代码中设置了 `pinMode(pin, INPUT_PULLUP)` |
| 继电器上电瞬间误动作 | 高电平有效导致复用前默认高电平触发 | 改为低电平有效，或在上电初始化时先拉高再配置 |
| RGB 亮度不均 | 各通道限流电阻未区分 | 按 4.3 节建议调整电阻值，或软件校准 PWM 占空比 |
