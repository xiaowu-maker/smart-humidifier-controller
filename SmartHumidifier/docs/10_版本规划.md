## V1.0 — 基础自动控制（MVP）

**代号**：Foundation
**目标**：实现本地智能湿度自动控制
**预计时间**：2026年7月-8月

### 功能清单

| 编号 | 功能 | 优先级 | 状态 |
|------|------|--------|------|
| F001 | ESP32 工程搭建 + 编译烧录 | P0 | 🔲 |
| F002 | AHT20 温湿度传感器驱动 | P0 | 🔲 |
| F003 | OLED 实时显示温湿度 | P0 | 🔲 |
| F004 | 继电器控制加湿器开关 | P0 | 🔲 |
| F005 | 自动滞回控制算法（45%-55%） | P0 | 🔲 |
| F006 | 三色LED状态指示 | P0 | 🔲 |
| F007 | 蜂鸣器异常报警 | P1 | 🔲 |
| F008 | 两个按钮（模式切换/恢复默认） | P1 | 🔲 |
| F009 | 串口日志输出 | P1 | 🔲 |
| F010 | 防抖和最小运行时间保护 | P0 | 🔲 |
| F011 | 72小时稳定性测试 | P0 | 🔲 |

### 技术栈

- PlatformIO + Arduino Framework
- ESP32 DevKit V1
- AHT20 (I2C)
- SSD1306 OLED (I2C)
- 无网络依赖

### 验收标准

- [ ] ESP32 成功启动并运行程序
- [ ] AHT20 正常读取温湿度（误差 ≤ ±2%RH, ±0.5℃）
- [ ] OLED 实时显示数据（刷新率 ≥ 0.5Hz）
- [ ] 继电器根据湿度自动控制加湿器
- [ ] 连续稳定运行 72 小时无异常
- [ ] 传感器断开时系统进入安全状态

## 版本发布规范

### 版本号规则

采用语义化版本号：`主版本.次版本.修订号`

| 变更类型 | 版本号 | 示例 |
|----------|--------|------|
| 不兼容的 API 修改 | 主版本 +1 | V1.0 → V2.0 |
| 向下兼容的功能新增 | 次版本 +1 | V1.0 → V1.1 |
| 向下兼容的问题修正 | 修订号 +1 | V1.0.0 → V1.0.1 |

### 每个版本的发布检查清单

- [ ] 所有功能测试通过
- [ ] 72小时稳定性测试通过
- [ ] CHANGELOG.md 更新
- [ ] README.md 更新
- [ ] API 文档更新（如有接口变更）
- [ ] 硬件兼容性确认
- [ ] Git Tag 打标签
- [ ] 固件二进制文件归档

### Git 分支策略

```
main          ← 稳定发布版本
  │
  ├── develop     ← 日常开发
  │     │
  │     ├── feature/v1.0-sensor    ← 功能分支
  │     ├── feature/v1.0-display
  │     └── feature/v1.0-control
  │
  └── release/v1.0    ← 发布准备
```

### Commit 规范

```
<type>(<scope>): <subject>

类型 (type)：
  feat     新功能
  fix      Bug修复
  docs     文档更新
  refactor 代码重构
  test     测试相关
  chore    构建/工具

示例：
  feat(sensor): add AHT20 driver with error handling
  fix(control): prevent relay flapping at threshold boundary
  docs(readme): add wiring diagram for OLED
```
