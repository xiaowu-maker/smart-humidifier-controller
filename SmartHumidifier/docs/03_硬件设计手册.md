## 1. 项目概述

SmartHumidifier 是一款基于 ESP32 微控制器的智能湿度监测与加湿控制装置。它通过 AHT20 温湿度传感器实时采集环境温湿度数据，在 0.96 寸 OLED 屏幕上直观显示，并可根据设定的湿度阈值自动控制加湿器的开关。此外，设备还配备了三色 LED 指示灯（显示湿度等级）和有源蜂鸣器（超限报警），以及两个物理按键用于用户交互。

本项目定位为入门级物联网硬件实践，旨在帮助初学者理解传感器数据采集、I2C 总线通信、继电器控制以及基本的嵌入式硬件设计流程。所有元器件均采用常见的开发板与模块，焊接工作量极少，核心连接均通过面包板和杜邦线完成。

## 3. 核心硬件详解

### 3.1 ESP32 DevKit V1 主控板

#### 概述

ESP32 DevKit V1 是乐鑫官方推荐的通用开发板，搭载 ESP-WROOM-32 模组，集成 WiFi（802.11 b/g/n）和蓝牙（v4.2 BR/EDR + BLE）双模通信。本项目的所有逻辑控制、数据处理和通信功能均由 ESP32 完成。

#### 核心规格参数

| 参数 | 值 |
|------|-----|
| 主控芯片 | ESP32-D0WDQ6（双核 Xtensa LX6，主频 240MHz） |
| Flash 闪存 | 4MB（SPI Flash） |
| SRAM | 520KB |
| 工作电压 | 5V（USB输入），3.3V（板载LDO稳压输出） |
| 逻辑电平 | 3.3V（GPIO输出高电平为3.3V） |
| 最大GPIO数量 | 25个（部分引脚有功能复用限制） |
| 通信接口 | UART × 3, SPI × 2, I2C × 2, I2S, CAN, 触摸传感器 × 10 |
| WiFi | 802.11 b/g/n，2.4GHz，HT40 |
| 蓝牙 | BR/EDR + BLE v4.2 |
| 板载LDO | AMS1117-3.3V，输出电流最大800mA |
| USB转串口芯片 | CP2102（或CH340G，视批次而定） |
| 尺寸 | 约 56 × 28 × 12mm |

#### 引脚分配表（本项目使用）

| 引脚 | 功能 | 连接目标 |
|:----:|------|----------|
| 3V3 | 3.3V电源输出 | 给AHT20、OLED供电 |
| 5V/VIN | 5V电源输入 | USB供电或VIN引脚 |
| GND | 公共地 | 所有模块共地 |
| GPIO21 (D21) | I2C数据线 (SDA) | AHT20 SDA + OLED SDA |
| GPIO22 (D22) | I2C时钟线 (SCL) | AHT20 SCL + OLED SCL |
| GPIO25 (D25) | 数字输出 | 继电器控制信号（IN） |
| GPIO26 (D26) | 数字输出 | RGB-LED 红色通道（R） |
| GPIO27 (D27) | 数字输出 | RGB-LED 绿色通道（G） |
| GPIO14 (D14) | 数字输出 | RGB-LED 蓝色通道（B） |
| GPIO12 (D12) | 数字输出 | 有源蜂鸣器控制 |
| GPIO13 (D13) | 数字输入（启用内部上拉） | 按键1（设置键） |
| GPIO15 (D15) | 数字输入（启用内部上拉） | 按键2（确认键） |

#### 引脚选型注意事项

1. **GPIO12 (D12)**：ESP32 的 GPIO12 在启动时是 Strapping 引脚，对应 MTDI。如果此引脚在启动时为高电平，会导致 Flash 电压被设置为 1.8V 模式，使模块无法正常启动。本项目中 GPIO12 用于蜂鸣器，默认状态下为低电平，因此安全可用。
2. **GPIO15 (D15)**：同样是 Strapping 引脚（MTDO），控制 UART0 的打印输出。默认有下拉电阻，此处用作按键输入，配置为内部上拉输入模式，不会影响启动。
3. **I2C 引脚**：ESP32 的 I2C 引脚可任意指定，但通常使用默认的 GPIO21（SDA）和 GPIO22（SCL），便于与 Arduino 库兼容。
4. **ADC2 引脚**：GPIO25、GPIO26、GPIO27 等属于 ADC2 通道，当 WiFi 启用时 ADC2 不可用。本项目中这些引脚仅用于数字输出，不受此限制影响。

### 3.2 AHT20 温湿度传感器

#### 概述

AHT20 是 ASAIR 公司推出的新一代温湿度传感器，采用 I2C 通信接口，出厂已校准，无需额外校准过程。相比旧款 DHT11/DHT22，AHT20 精度更高、响应更快，且使用标准的 I2C 总线，便于与 OLED 等设备共享总线。

#### 核心规格参数

| 参数 | 值 |
|------|-----|
| 型号 | AHT20 |
| 通信接口 | I2C（最大速率 400kHz） |
| I2C 设备地址 | **0x38**（7位地址） |
| 温度测量范围 | -40℃ ~ +85℃ |
| 温度精度 | ±0.3℃（典型值，在 25℃ 时） |
| 湿度测量范围 | 0% ~ 100% RH |
| 湿度精度 | ±2% RH（典型值，在 25℃ 时） |
| 分辨率 | 温度 0.01℃，湿度 0.024% RH |
| 响应时间 | 典型值 < 5 秒 |
| 工作电压 | 2.2V ~ 5.5V |
| 工作电流 | 测量时约 0.5mA，休眠时约 0.1μA |
| 模块尺寸 | 约 13 × 13mm（不含排针） |

#### 引脚定义

| 引脚 | 名称 | 功能 | 连接目标 |
|:----:|:----:|------|----------|
| 1 | VCC | 电源正极（2.2~5.5V） | ESP32 3V3 |
| 2 | SCL | I2C 时钟线 | ESP32 GPIO22 (SCL) |
| 3 | SDA | I2C 数据线 | ESP32 GPIO21 (SDA) |
| 4 | GND | 电源地 | ESP32 GND |

> **注意**：部分模块标记为 VDD/VSS 或 +/SDA/SCL/-，功能对应关系不变。AHT20 的 I2C 地址固定为 0x38，不可更改。该地址与 OLED 的 0x3C 不冲突。

#### 工作原理简述

AHT20 内部包含一个电容式湿度传感器和一个带隙温度传感器。测量时，主机通过 I2C 发送 0xAC 启动测量命令，等待约 80ms 后读取 6 字节数据，其中湿度为 20 位原始值，温度为 20 位原始值，通过公式转换得到实际温湿度。

### 3.3 OLED 0.96寸显示屏 (SSD1306)

#### 概述

0.96 寸 OLED 显示屏采用 SSD1306 驱动芯片，单色（白色/蓝色/黄色，视型号而定），分辨率为 128×64 像素。通过 I2C 接口与主控通信。OLED 为自发光显示，不需要背光，对比度高，功耗低，非常适合嵌入式项目。

#### 核心规格参数

| 参数 | 值 |
|------|-----|
| 驱动芯片 | SSD1306 |
| 分辨率 | 128 × 64 像素 |
| 显示颜色 | 白色 / 蓝色（根据型号） |
| 通信接口 | I2C（支持 400kHz Fast Mode） |
| I2C 设备地址 | **0x3C**（7位地址） |
| 工作电压 | 3.3V ~ 5V |
| 工作电流 | 约 20mA（全亮） |
| 像素尺寸 | 0.96 英寸（对角线） |
| 可视角度 | > 160° |
| 模块尺寸 | 约 27 × 27 × 4mm |

#### 引脚定义

| 引脚 | 名称 | 功能 | 连接目标 |
|:----:|:----:|------|----------|
| 1 | VCC / VDD | 电源正极（3.3~5V） | ESP32 3V3 |
| 2 | GND / VSS | 电源地 | ESP32 GND |
| 3 | SCL / SCK | I2C 时钟线 | ESP32 GPIO22 (SCL) |
| 4 | SDA / MOSI | I2C 数据线 | ESP32 GPIO21 (SDA) |

> **提示**：部分 OLED 模块可能有第5脚 RES（复位）或第6脚 DC（数据/命令），在 I2C 模式下无需连接。如果模块还有 CS（片选）引脚，也无需连接。购买时请选择 "I2C 接口" 版本而非 SPI 版本。

### 3.4 单路继电器模块

#### 概述

继电器是一种电控机械开关，通过低电压小电流控制高电压大电流的通断。本项目使用单路 5V 继电器模块来控制加湿器的 220V 交流电源。ESP32 的 GPIO25 输出高/低电平信号，经继电器模块上的 S8050 三极管驱动，控制继电器线圈的通电与断电，从而控制加湿器的插孔通断。

#### 核心规格参数

| 参数 | 值 |
|------|-----|
| 驱动电压 | 5V DC |
| 驱动电流 | 约 70mA（线圈吸合时） |
| 触点额定值 | 250VAC / 10A（最大）；30VDC / 10A |
| 控制信号电平 | 3.3V / 5V 兼容（高电平触发） |
| 吸合时间 | 约 10ms |
| 释放时间 | 约 5ms |
| 机械寿命 | 约 10,000,000 次 |
| 电气寿命 | 约 100,000 次（额定负载） |
| 模块尺寸 | 约 25 × 23 × 18mm |

#### 引脚定义

| 引脚/端子 | 名称 | 功能 | 连接目标 |
|:----------:|:----:|------|----------|
| DC+ | 电源正 | 5V 供电正极 | ESP32 5V（或外接5V） |
| DC- | 电源地 | 供电负极 | ESP32 GND |
| IN | 控制信号输入 | 高电平触发吸合 | ESP32 GPIO25 |
| COM | 公共端 | 继电器公共触点 | 接加湿器电源的火线(L)入 |
| NO | 常开端 | 常开触点（默认断开） | 接加湿器火线出（到加湿器） |
| NC | 常闭端 | 常闭触点（默认闭合） | 本项目中不使用 |

> **工作逻辑说明**：
> - 当 IN 引脚为 **高电平** 时：继电器吸合，COM 与 NO 导通，加湿器通电工作。
> - 当 IN 引脚为 **低电平** 时：继电器释放，COM 与 NO 断开，加湿器断电停止。
> - 注意：部分低电平触发模块（标记为 LOW Trigger）则逻辑相反，购买时请确认。

#### 接线示意（继电器端）

```
      ┌──────────────┐
      │              │
      │   ┌────┐    │
      │   │ ~  │    │  ← 继电器指示灯
      │   └────┘    │
      │              │
      │  DC+ DC- IN │  ← 控制侧：接 ESP32
      │   │   │   │  │
      │   └───┴───┘  │
      │              │
      │  COM  NO  NC │  ← 负载侧：接220V
      │   │    │    │  │
      │   └────┴────┘  │
      └──────────────┘
```

### 3.5 三色LED (共阴极RGB)

#### 概述

RGB LED 将红色（R）、绿色（G）、蓝色（B）三个 LED 芯片封装在同一个外壳中。共阴极类型意味着三个 LED 的负极公共端连接在一起，接 GND；三个正极分别通过限流电阻连接到 ESP32 的 GPIO 引脚，高电平点亮。

#### 核心规格参数

| 参数 | 值 |
|------|-----|
| 类型 | 共阴极 RGB LED |
| 外形 | 5mm 圆头 / 方头 |
| 正向电压（R） | 约 2.0V |
| 正向电压（G/B） | 约 3.0V |
| 推荐工作电流 | 10~20mA（每通道） |
| 限流电阻计算 | R = (3.3V - Vf) / 15mA |

#### 引脚定义及限流电阻计算

| 引脚 | 名称 | 功能 | 限流电阻 | 连接目标 |
|:----:|:----:|------|:---------:|----------|
| 最长脚 | GND / 公共阴极 | 共地 | — | ESP32 GND |
| 其他脚（一） | R / 红色阳极 | 红色通道控制 | 220Ω | ESP32 GPIO26 |
| 其他脚（二） | G / 绿色阳极 | 绿色通道控制 | 220Ω | ESP32 GPIO27 |
| 其他脚（三） | B / 蓝色阳极 | 蓝色通道控制 | 220Ω | ESP32 GPIO14 |

**电阻计算验证**（以红色通道为例）：

- 红色 LED 正向压降 Vf ≈ 2.0V
- ESP32 GPIO 输出高电平 ≈ 3.3V
- 目标电流 I ≈ 15mA
- 限流电阻 R = (3.3V - 2.0V) / 0.015A ≈ 86.7Ω

实际上选择 220Ω 更为保守（电流约 6mA），亮度足够且不会过流。绿色和蓝色通道 Vf ≈ 3.0V，使用 220Ω 时电流约 1.4mA，亮度略低但仍可见。如果希望三色亮度一致，可以在红色通道使用 100Ω，绿色和蓝色通道使用 220Ω。

#### 颜色控制表

| 状态 | R (GPIO26) | G (GPIO27) | B (GPIO14) | 含义 |
|:----:|:----------:|:----------:|:----------:|------|
| 绿色 | LOW | HIGH | LOW | 湿度正常 |
| 蓝色 | LOW | LOW | HIGH | 湿度过低 |
| 红色 | HIGH | LOW | LOW | 湿度过高 |
| 黄色 | HIGH | HIGH | LOW | 设备初始化中 |
| 紫色 | HIGH | LOW | HIGH | 报警状态 |
| 白色 | HIGH | HIGH | HIGH | 全亮（测试模式） |
| 熄灭 | LOW | LOW | LOW | 待机/关闭 |

### 3.6 有源蜂鸣器

#### 概述

有源蜂鸣器内部包含振荡电路，通电即发出固定频率的声音（通常约 2~4kHz），只需提供直流电压即可驱动。与无源蜂鸣器（需 PWM 信号驱动）不同，有源蜂鸣器控制简单，适合用作报警提示。

#### 核心规格参数

| 参数 | 值 |
|------|-----|
| 类型 | 有源蜂鸣器（含振荡电路） |
| 额定电压 | 5V |
| 工作电压 | 3.3V ~ 5V |
| 额定电流 | 约 25~40mA（5V时） |
| 发声频率 | 约 2300 ± 300Hz（固定） |
| 声压级 | ≥ 85dB（10cm处典型值） |
| 外形 | 直径约 12mm，高约 9.5mm |

#### 引脚定义

| 引脚 | 名称 | 功能 | 连接目标 |
|:----:|:----:|------|----------|
| 长脚 / + | VCC (+) | 电源正极 | ESP32 GPIO12 |
| 短脚 / - | GND (-) | 电源地 | ESP32 GND |
| 中间脚 | — | 部分型号无此脚 | 不连接 |

> **注意**：
> 1. 蜂鸣器工作电流较大（25~40mA），ESP32 的单个 GPIO 最大输出电流有限（约 12mA），因此**建议通过 NPN 三极管（如 S8050）或使用 ESP32 的 GPIO 控制一个 MOS 管来驱动**。简单做法：可使用 GPIO12 直接驱动（风险较低，因为实际使用时蜂鸣器不可能长时间连续工作），但更稳妥的方式是通过 2N2222 或 S8050 三极管搭建驱动电路。
> 2. 蜂鸣器有正负极之分，接反不会发声。

#### 三极管驱动电路（推荐）

```
   ESP32 GPIO12 ──┬── 1kΩ ── R ──┐
                  │                │
                  C               B (2N2222/S8050 NPN)
                 ┌┴┐              │
                 │ │              E
                 │ │              │
                 └┬┘              │
                  │               │
                  │1N4148         GND
                  │ (续流二极管)
  蜂鸣器 (+) ────┤
  蜂鸣器 (-) ────┴──── GND
```

如果不使用三极管，也可直接将蜂鸣器接在 ESP32 的 VCC（5V）和 GPIO 之间（GPIO 输出低电平时拉通），但这会使电流流过 GPIO 的灌电流路径，不建议初学者采用。**本项目的简化接线方案中，采用 GPIO12 直接驱动蜂鸣器正极**，适用于短时间报警场景，使用时长请控制在 5 秒内。

### 3.7 按键 (Tact Switch)

#### 概述

轻触按键（Tact Switch）是最基础的人机交互元件。按下时内部两个触点导通，松开后自动弹开。本项目使用两个按键：
- **按键1（SET**）：模式切换 / 进入设置 / 数值增加
- **按键2（OK**）：确认选择 / 确认报警

#### 核心规格参数

| 参数 | 值 |
|------|-----|
| 型号 | 6×6mm 卧式或立式轻触开关 |
| 触点电阻 | ≤ 100mΩ（按下时） |
| 绝缘电阻 | ≥ 100MΩ（释放时） |
| 使用寿命 | ≥ 100,000 次 |
| 按压力 | 约 160 ± 50gf |
| 引脚 | 四脚（两两相通，对边连通） |

#### 按键的接线方式

按键需要配合**上拉电阻**或**下拉电阻**使用，否则引脚悬浮会导致逻辑电平不稳定。本项目采用 ESP32 的内部上拉电阻（通过 `pinMode(pin, INPUT_PULLUP)` 启用），无需外部电阻。

```
   ┌──── 3.3V (通过ESP32内部上拉)
   │
   ╱
   ╲  R_pullup ≈ 45kΩ (内部)
   ╱
   │
   ├────┐ ┌──── ESP32 GPIO
   │    │ │
   │  ──┘ ├── 按键 (Tact Switch)
   │      │
  GND ────┘

   未按下：GPIO 读为 HIGH
   按下时：GPIO 与 GND 导通，读为 LOW
```

#### 引脚定义（结合内部上拉）

| 按键 | 引脚连接 | ESP32 GPIO | 配置 |
|:----:|----------|:----------:|------|
| 按键1 | 一脚接 GND，另一脚接 GPIO | GPIO13 | INPUT_PULLUP |
| 按键2 | 一脚接 GND，另一脚接 GPIO | GPIO15 | INPUT_PULLUP |

> **消抖处理**：机械按键在按下和释放的瞬间会有电平抖动（通常持续 10~50ms），建议在代码中使用 50ms 延时去抖或状态机去抖算法。

## 4. I2C 总线设计

I2C（Inter-Integrated Circuit）是一种同步、多主从、串行通信总线，仅需两根信号线（SDA 数据线、SCL 时钟线）即可挂载多个设备。

### 4.1 总线拓扑

本项目中，AHT20 和 OLED 两个 I2C 设备共享同一组 SDA/SCL 总线，连接至 ESP32 的 GPIO21（SDA）和 GPIO22（SCL）。

```
                      ESP32
                   ┌─────────┐
                   │ GPIO21  ├─────┬────────────── SDA 数据线
                   │ GPIO22  ├──┬──┴────────────── SCL 时钟线
                   └─────────┘  │
                                │
                ┌───────────────┼───────────────┐
                │               │               │
           ┌────┴────┐    ┌────┴────┐      ┌────┴────┐
           │  AHT20  │    │  OLED   │      │  (预留)  │
           │ 0x38    │    │ 0x3C    │      │         │
           │ SDA SCL │    │ SDA SCL │      │         │
           └────┬────┘    └────┬────┘      └─────────┘
                │              │
                └──────┬──────┘
                       │
                     ┌─┴─┐  ┌─┴─┐
                     │ R │  │ R │  4.7kΩ 上拉电阻
                     └─┬─┘  └─┬─┘
                       │      │
                     3.3V   3.3V
```

### 4.2 I2C 地址确认

| 设备 | 7位地址 | 写地址 (<<1) | 读地址 | 冲突检查 |
|:----:|:-------:|:------------:|:------:|:--------:|
| AHT20 | 0x38 | 0x70 | 0x71 | 不冲突 ✓ |
| OLED (SSD1306) | 0x3C | 0x78 | 0x79 | 不冲突 ✓ |

> **注意**：如果 I2C 总线上有两个设备地址相同，将导致通信冲突，两个设备均无法正常工作。本项目所选的两个设备地址不同，可以安全共享总线。

### 4.3 上拉电阻

I2C 总线要求 SDA 和 SCL 线路上有上拉电阻，将空闲总线拉到高电平。
- 标准 I2C 总线规范推荐上拉电阻值为 4.7kΩ（100kHz 模式）或 2.2kΩ（400kHz 模式）。
- ESP32 的某些 I2C 引脚内部已集成上拉电阻（约 45kΩ），但强度太弱，需要外部上拉。
- **重要**：市面上大多数 AHT20 和 OLED 模块都已自带 4.7kΩ 上拉电阻（焊接在 PCB 上）。如果购买的是模块而非裸芯片，通常不必额外添加上拉电阻。如果在总线上串联了很长的杜邦线（>20cm），可考虑在 ESP32 侧再并一个 4.7kΩ 电阻以增强抗干扰能力。

### 4.4 I2C 电气连接注意事项

1. **总线长度**：I2C 总线的连线总长度建议控制在 50cm 以内，本项目在面包板上搭建，通常不超过 20cm，完全满足要求。
2. **总线电容**：每增加一个设备会引入额外的引脚电容，过长的杜邦线也会增加线间电容，可能导致信号边沿变缓。如需延长，可降低 I2C 速率（在 Arduino 中使用 `Wire.setClock(100000)` 设置为 100kHz）。
3. **地址冲突检查**：上电后可通过 I2C 扫描程序快速验证总线上的设备。Arduino 示例：

```cpp
#include <Wire.h>
void setup() {
  Wire.begin();
  Serial.begin(115200);
  Serial.println("I2C Scanner");
  for (byte addr = 1; addr < 127; addr++) {
    Wire.beginTransmission(addr);
    if (Wire.endTransmission() == 0) {
      Serial.print("Found device at 0x");
      Serial.println(addr, HEX);
    }
  }
}
void loop() {}
```

期望输出：
```
Found device at 0x38   // AHT20
Found device at 0x3C   // OLED
```

## 6. 电源设计详解

### 6.1 供电拓扑

```
  ┌─────────────────────────────┐
  │    5V / 2A USB 电源适配器    │
  │         (AC 220V → DC 5V)   │
  └──────────────┬──────────────┘
                 │
          Micro USB 线
                 │
  ┌──────────────┴──────────────────────┐
  │        ESP32 DevKit V1              │
  │                                     │
  │  USB 端口 (5V)                      │
  │     │                               │
  │     ├── 保护二极管 ──┬── VBUS (5V)  │
  │     │               │              │
  │     │               ├── VIN排针 ────┼──→ 继电器 DC+ (5V)
  │     │               │              │
  │     │               └── AMS1117    │
  │     │                    │ 3.3V    │
  │     │                    │         │
  │     │                    ├── 3V3排针 ──→ AHT20 VCC
  │     │                    │            → OLED VCC
  │     │                    │            → 三色LED (通过GPIO)
  │     │                    │            → 内部上拉
  │     │                    │            → ESP32 内部电路
  │     │                    │
  │     └────────────────────┴── GND ─────→ 所有模块共地
  │                                     │
  └─────────────────────────────────────┘
```

### 6.2 稳压设计

ESP32 DevKit V1 板载的 AMS1117-3.3V 线性稳压器将 USB 输入的 5V 转换为 3.3V，为本体及外设供电。

**AMS1117-3.3 特性**：
- 输出电压：3.3V ± 1.5%
- 最大输出电流：800mA
- 压差（Dropout Voltage）：典型 1.1V（满载时 1.3V）
- 过流保护、过热保护

由于 3.3V 侧总电流需求约 107mA（详见电源容量计算），AMS1117 完全在安全范围内工作。其线性稳压效率约为 3.3/5 = 66%，多余的能量以热量形式耗散。在 107mA 负载下，功耗约 (5-3.3)V × 0.107A = 0.18W，温升很小。

### 6.3 电源注意事项

1. **继电器供电**：继电器线圈需要 5V 驱动电流约 70mA，直接从 ESP32 的 VIN 引脚获取（VIN 直通 USB 5V）。**不要使用 3.3V 给继电器供电**，可能导致吸合不可靠或无法吸合。
2. **蜂鸣器供电**：蜂鸣器额定 5V，但直接由 GPIO12 以 3.3V 驱动时仍然可以发声（声音稍小）。如需更大音量，可将蜂鸣器正极接 5V（串联 100Ω 电阻限流），负极通过 NPN 三极管到 GND。
3. **电源滤波**：如果观察到 OLED 屏幕闪烁或传感器数据跳变，可能是电源纹波过大。可在面包板电源轨上并联一个 100μF 电解电容（正极接 VCC，负极接 GND）和一个 0.1μF 瓷片电容进行滤波。
4. **杜绝反接**：所有模块的 VCC 和 GND 必须确认后再通电。反接可能永久损坏模块。
5. **热插拔风险**：不建议在通电状态下插拔模块，尤其不要插拔 OLED 和 AHT20（I2C 设备热插拔可能导致总线锁死）。

### 6.4 可选：独立 3.3V 稳压方案

如果后续需要扩展更多 3.3V 外设（如 SD 卡、更多传感器），可考虑在面包板上增加独立的 AMS1117-3.3V 模块，分担 ESP32 板载稳压器的负载：

```
  USB 5V ──┬── ESP32 VIN
            │
            ├── 独立 AMS1117-3.3 输入
            │        │
            │        └── 输出 3.3V → 外设
            │
            └── 继电器 DC+ (5V)
```

本项目暂不需要。

## 8. 硬件组装步骤

以下为建议的组装顺序，请逐步操作并在每一步完成后检查确认。

### 第1步：准备与检查（约5分钟）

```
□ 确认所有硬件已到齐（对照硬件清单）
□ 使用万用表电阻档检查面包板电源轨是否导通（红笔接红色轨一端，黑笔接另一端）
□ 检查面包板电源轨正负是否短路（红笔接红轨，黑笔接蓝轨，应无穷大）
□ 目视检查各模块PCB有无明显损坏、焊点脱落
□ 准备杜邦线，按颜色分类
```

### 第2步：安装ESP32到面包板（约3分钟）

```
1. 将 ESP32 DevKit V1 轻轻插入面包板中央凹槽的一侧
2. ESP32 应跨越凹槽安装，排针插入凹槽两侧的标准插孔中
3. 确保排针对齐，不要斜着插入
4. 注意用力均匀，避免排针弯曲

   ┌──────────────────────────────┐
   │         面包板                │
   │  ┌────────────────────────┐  │
   │  │  ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ │  │  ← 排针插入这一列
   │  │                        │  │
   │  │═══════════凹槽══════════│  │  ← 中央凹槽
   │  │                        │  │
   │  │  ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ │  │  ← 排针插入这一列
   │  └────────────────────────┘  │
   │    ESP32 开发板               │
   └──────────────────────────────┘
```

> **提示**：ESP32 DevKit V1 插在面包板上会占据约 8~10 行每侧的空间。布局时靠面包板一侧放置，为其他模块留出空间。

### 第3步：连接电源总线（约3分钟）

```
1. 用红色杜邦线（公对母），一端接 ESP32 的 3V3 排针，另一端插入面包板红色电源轨的某一行
2. 用黑色杜邦线（公对母），一端接 ESP32 的 GND 排针，另一端插入面包板蓝色电源轨的某一行
3. （可选）用红色杜邦线，一端接 ESP32 的 VIN/5V 排针，另一端插入另一组红色电源轨（用于继电器供电）
4. 确认：万用表直流电压档测量电源轨，应为 3.3V 和 0V（GND）
```

### 第4步：安装AHT20传感器（约5分钟）

```
1. 将 AHT20 模块插入面包板中部（确保不与其他模块冲突的位置）
2. VCC ← 红色杜邦线 → 面包板红色电源轨（3.3V）
3. GND ← 黑色杜邦线 → 面包板蓝色电源轨（GND）
4. SDA ← 绿色杜邦线 → 插入某个空行（标记为 SDA_BUS）
5. SCL ← 黄色杜邦线 → 插入某个空行（标记为 SCL_BUS）
```

### 第5步：安装OLED显示屏（约5分钟）

```
1. 将 OLED 模块插入面包板（与 AHT20 保持适当距离）
2. VCC ← 红色杜邦线 → 面包板红色电源轨（3.3V）
3. GND ← 黑色杜邦线 → 面包板蓝色电源轨（GND）
4. SDA ← 绿色杜邦线 → 与 AHT20 SDA 相同的行（SDA_BUS）
5. SCL ← 黄色杜邦线 → 与 AHT20 SCL 相同的行（SCL_BUS）
```

### 第6步：建立 I2C 总线（约2分钟）

```
1. 从 SDA_BUS 行（AHT20 SDA 和 OLED SDA 所在的同一行）← 杜邦线 → ESP32 GPIO21
2. 从 SCL_BUS 行（AHT20 SCL 和 OLED SCL 所在的同一行）← 杜邦线 → ESP32 GPIO22
3. 确认：I2C 总线上两个设备地址不冲突（0x38 和 0x3C），已无外部上拉电阻需要添加
```

### 第7步：安装三色LED（约5分钟）

```
1. 将 RGB LED 插入面包板，确保四个引脚分别位于不相通的行
2. 公共阴极（最长脚）← 黑色杜邦线 → 面包板蓝色电源轨（GND）
3. R 脚 ← 220Ω 电阻 → 杜邦线 → ESP32 GPIO26
4. G 脚 ← 220Ω 电阻 → 杜邦线 → ESP32 GPIO27
5. B 脚 ← 220Ω 电阻 → 杜邦线 → ESP32 GPIO14

   LED引脚识别（平面朝下，面向自己）：
        ┌────┐
        │    │
        │ R  │  ← 最左边有缺角标记处为 R
        │ G  │
        │ B  │  ← B
        │ G  │  ← 公共阴极（最长脚，中间偏右）
        └────┘
```

### 第8步：安装蜂鸣器（约3分钟）

```
1. 蜂鸣器插入面包板
2. 正极 (+) ← 杜邦线 → ESP32 GPIO12
3. 负极 (-) ← 黑色杜邦线 → 面包板蓝色电源轨（GND）
```

### 第9步：安装按键（约5分钟）

```
1. 按键1（SET）插入面包板
   - 一脚 ← 杜邦线 → ESP32 GPIO13
   - 另一脚 ← 杜邦线 → 面包板蓝色电源轨（GND）
2. 按键2（OK）插入面包板
   - 一脚 ← 杜邦线 → ESP32 GPIO15
   - 另一脚 ← 杜邦线 → 面包板蓝色电源轨（GND）
3. 确认：使用万用表通断档，按下按键时对应 GPIO 引脚与 GND 导通
```

### 第10步：安装继电器模块（约3分钟）

```
1. 继电器模块（控制侧）插入面包板
   - DC+ ← 红色杜邦线 → 面包板红色电源轨（5V/VIN，不是3.3V）
   - DC- ← 黑色杜邦线 → 面包板蓝色电源轨（GND）
   - IN  ← 杜邦线 → ESP32 GPIO25
2. 确认：用万用表直流电压档测 DC+ 与 DC- 之间电压，应为 4.5~5.5V
```

### 第11步：通电前检查（约5分钟）

此步骤至关重要，请逐项确认：

```
□ 电源检查
   □ 红色电源轨（3.3V）与蓝色电源轨（GND）之间无短路
   □ 红色电源轨（5V）与蓝色电源轨（GND）之间无短路
   □ ESP32 的 3V3 与 GND 排针之间无短路
   
□ 模块上电检查
   □ AHT20 VCC 与 GND 之间无短路
   □ OLED VCC 与 GND 之间无短路
   □ 继电器 DC+ 与 DC- 之间无短路
   
□ 无电源反接
   □ 所有模块 VCC 均接电源正极（3.3V 或 5V）
   □ 所有模块 GND 均接电源地
   
□ I2C 总线检查
   □ SDA 线未连接到 VCC 或 GND
   □ SCL 线未连接到 VCC 或 GND
   □ SDA 和 SCL 之间未短路
   
□ 继电器检查
   □ 220V 侧 COM/NO/NC 未连接到任何面包板电路（隔离）
   □ 如果已接220V（不推荐此时接），确保线路完整绝缘
   
□ 整体检查
   □ 有线无松动、无裸露线头
   □ 面包板插孔无金属碎屑
   □ 杜邦线插头完全插入
```

### 第12步：通电测试（约5分钟）

```
1. 将 Micro USB 线连接到 ESP32
2. 另一端连接电脑 USB 口（或 5V 适配器）
3. 观察 ESP32 板载 LED（PWR 指示灯）应该亮起
4. 观察 OLED 屏幕（在程序烧录前可能不显示，正常）
5. 用手触碰 AHT20，感受是否需要预热（通电后传感器需要约 1 秒稳定时间）
6. 如果发现任何异常（异味、冒烟、模块发烫），立即断开电源
```

### 第13步：上传测试程序（约10分钟）

完成硬件连接后，请上传一个简单的测试程序验证各模块功能。以下为 Arduino 框架的引脚通断测试代码框架（仅用于验证硬件连接是否正常）：

```cpp
// ===== 硬件测试伪代码 =====
// 此代码用于硬件连通性测试，非最终产品代码

void setup() {
  // 配置所有输出引脚
  pinMode(25, OUTPUT); // 继电器
  pinMode(26, OUTPUT); // LED R
  pinMode(27, OUTPUT); // LED G
  pinMode(14, OUTPUT); // LED B
  pinMode(12, OUTPUT); // 蜂鸣器
  pinMode(13, INPUT_PULLUP); // 按键1
  pinMode(15, INPUT_PULLUP); // 按键2
  
  // I2C 初始化
  Wire.begin();
  Serial.begin(115200);
}

void loop() {
  // 测试1：RGB LED 颜色循环（每个颜色亮 1 秒）
  digitalWrite(26, HIGH); delay(1000); digitalWrite(26, LOW); // 红
  digitalWrite(27, HIGH); delay(1000); digitalWrite(27, LOW); // 绿
  digitalWrite(14, HIGH); delay(1000); digitalWrite(14, LOW); // 蓝
  
  // 测试2：继电器吸合 2 秒
  digitalWrite(25, HIGH); delay(2000); digitalWrite(25, LOW); delay(1000);
  
  // 测试3：蜂鸣器响 1 秒
  digitalWrite(12, HIGH); delay(1000); digitalWrite(12, LOW); delay(1000);
  
  // 测试4：读取按键状态到串口
  Serial.print("Key1: "); Serial.print(digitalRead(13));
```

## 9. 安全注意事项

**安全是硬件设计的首要原则。** 本项目中涉及到 220V 交流电的继电器控制，操作不当可能导致触电、火灾甚至人身伤害。请务必遵守以下安全规范。

### 9.1 低压侧安全（面包板电路，电压 ≤ 5V）

低压侧的风险相对较低，但仍需注意：

| 风险 | 说明 | 预防措施 |
|:----|:----|----------|
| 短路 | 杜邦线裸露金属部分接触导致供电短路 | 使用万用表通断档检查电源轨是否短路后再通电 |
| 反接 | 模块 VCC/GND 接反 | 通电前逐模块检查，红色线接 VCC，黑色线接 GND |
| 过流 | GPIO 输出电流超过额定值 | 使用限流电阻（LED必须串联220Ω电阻） |
| 过热 | 稳压器长时间工作在超额定电流 | 不超出总电流预算；必要时增加散热 |
| GPIO损坏 | 误将 GPIO 接 VCC 或 GND 长时间 | 避免引脚短路；输出引脚不相互连接 |

**低压侧黄金法则**：
- 先接线，后通电；先检查，后通电。
- 通电时如果模块异常发热（>60℃），立即断开电源。
- 不要在通电时插拔杜邦线。
- 使用万用表"通断档"检查疑似短路的位置时，电路必须断电。

### 9.2 ⚠️ 高压侧安全（继电器 220V 侧）

**这是本项目的最高风险区域。** 请仔细阅读并遵守以下每一条：

#### 风险等级

```
220V 交流电风险：
┌──────────────────────────────────────────┐
│  ☠ 触电 - 可致命   ⚠ 火灾 - 财产损失     │
│  ⚡ 电弧 - 烧伤      🔥 过热 - 绝缘损坏    │
└──────────────────────────────────────────┘
```

#### 安全规范

1. **专业分工**：如果对 220V 电气接线不熟悉，请找有电工证的专业人士协助完成继电器负载侧的接线。
2. **断电操作**：任何接线操作前，**必须断开 220V 电源**（拔掉插头或关闭空气开关）。
3. **绝缘处理**：所有 220V 接线点必须使用绝缘胶带或热缩管包裹，确保无裸露铜线。
4. **保险丝保护**：在加湿器供电回路中建议串联一个 5A / 250V 的保险丝座 + 保险丝管，一旦发生过流会自动熔断。
5. **防潮防水**：加湿器工作环境湿度较高，继电器模块和 220V 接点必须远离水滴，建议安装在密封电控盒中。
6. **物理隔离**：面包板上的低压电路与继电器 220V 接线必须在物理上分离。推荐使用绝缘端子台进行 220V 接线。
7. **接地保护**：如果使用金属外壳，外壳必须可靠接地。
8. **急停开关**：建议在 220V 回路中串联一个带灯急停开关，方便紧急断电。

#### 接线图示（220V 侧）

```
             ┌── L (火线，通常棕色或红色)
             │
        ┌────┤ 电源插头
        │    │
        │    └── N (零线，通常蓝色或黑色)
        │
    [5A 保险丝]
        │
        │
    ┌───┤ 继电器模块
    │   │
    │ COM        NO ──── 加湿器电源输入的火线
    │   │                │
    │   │              ┌─┴─┐
    │   │              │   │
    │   │              │加 │  加湿器
    │   │              │湿 │
    │   │              │器 │
    │   │              │   │
    │   │              └─┬─┘
    │   │                │
    │   └────────────────┴── 加湿器电源输入的零线
    │
    └─────────────── N (零线直通)
```

**另一种更安全的方案**：使用**成品射频遥控插座**改装，将继电器的触点并联到遥控插座的按钮两端，避免直接处理 220V。此方案适合初学者，更安全。

### 9.3 ESD 静电防护

- 在干燥环境中手指可能积累数千伏静电，足以损坏 CMOS 芯片（如 ESP32、AHT20）。
- 在接触模块前，先触摸接地的金属物体（如电脑机箱）释放静电。
- 尽量避免直接触摸芯片的引脚，握住 PCB 板边缘操作。

### 9.4 使用环境要求

| 参数 | 建议范围 | 说明 |
|:----|:---------|------|
| 工作温度 | 0℃ ~ 50℃ | 超出范围可能导致传感器精度下降 |
| 工作湿度 | 20% ~ 85% RH（非凝结） | 高湿度可能导致电路板漏电 |
| 放置 | 远离水源、暖气、阳光直射 | 避免影响湿度测量和电气安全 |
| 通风 | 继电器模块需留散热空间 | 长时间吸合时线圈会发热 |

## 11. 常见问题与排查

### 11.1 上电后无反应

| 可能原因 | 排查方法 | 解决 |
|:---------|:---------|------|
| USB线不通 | 换一根手机数据线（确保带数据传输功能） | 充电线可能只有电源无数据线 |
| 电源适配器故障 | 用万用表测USB口输出电压 | 换一个适配器 |
| ESP32损坏 | 板载PWR灯不亮 | 更换ESP32开发板 |
| 面包板电源轨断路 | 万用表测ESP32 3V3到面包板电源轨 | 重插杜邦线 |

### 11.2 I2C 扫描不到设备

| 可能原因 | 排查方法 | 解决 |
|:---------|:---------|------|
| AHT20 或 OLED 未供电 | 万用表 DC 档测模块 VCC/GND | 确认模块在 3.3V 电源轨上 |
| SDA/SCL 接反 | 检查 SDA 是否接 GPIO21，SCL 接 GPIO22 | 交换两者 |
| 模块损坏 | 换一个模块测试 | 用备用模块替换 |
| 总线锁死 | SDA 一直为低电平 | 重新上电（热插拔可能锁死 I2C 总线） |
| 地址错误 | AHT20 地址是 0x38，OLED 是 0x3C | 检查代码中 I2C 地址是否正确 |

### 11.3 OLED 不显示

| 可能原因 | 排查方法 | 解决 |
|:---------|:---------|------|
| I2C 连线问题 | 确认 SDA/SCL 接线正确 | 重接 |
| 地址错误 | 库中默认地址可能是 0x3D | 改为 0x3C |
| 对比度设置 | 上电后 OLED 可能全灭 | 调用 `display.setContrast()` |
| 模块损坏 | 换新模块 | 替换 |

### 11.4 AHT20 读数异常

| 可能原因 | 排查方法 | 解决 |
|:---------|:---------|------|
| 传感器预热不足 | 首次上电前 1 秒数据可能不准 | 等待 2 秒后再读数 |
| I2C 速率太高 | 超过 400kHz 可能导致通信错误 | `Wire.setClock(100000)` 降低速率 |
| 电源纹波大 | OLED 全亮时采样干扰 | 添加 100μF 电容滤波 |
| 传感器损坏 | 读取恒定值 0xFFFFFFFF | 更换传感器 |

### 11.5 继电器不动作

| 可能原因 | 排查方法 | 解决 |
|:---------|:---------|------|
| 供电电压不足 | 测 DC+/DC- 电压 < 4.5V | 使用5V电源轨（非3.3V） |
| IN 信号错误 | GPIO25 是否输出 HIGH | 检查代码逻辑 |
| 三极管驱动管损坏 | 继电器模块指示灯不亮 | 更换模块 |
| 模块是高电平触发还是低电平 | 确认买到的是 HIGH Trigger | LOW Trigger 需反转代码逻辑 |

### 11.6 RGB LED 亮度不一致

| 可能原因 | 排查方法 | 解决 |
|:---------|:---------|------|
| 正向压降不同 | R/G/B 的 Vf 不同 | 调整各通道限流电阻值 |
| GPIO 驱动能力差异 | 各 GPIO 输出略有差异 | 使用 PWM 调亮度平衡 |
