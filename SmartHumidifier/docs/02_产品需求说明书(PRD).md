## 2. 产品背景与战略定位

### 2.1 行业背景

随着居民生活水平的提升，室内环境质量日益受到关注。研究表明，人体舒适的相对湿度范围为 **45%~55%RH**。湿度过低（<30%RH）会导致皮肤干燥、呼吸道不适、静电增加等问题；湿度过高（>70%RH）则容易滋生霉菌、尘螨，诱发呼吸系统疾病。

当前市场上，传统加湿器存在以下痛点：

1. **缺乏智能闭环控制**：多数产品需要用户手动开关，无法根据环境湿度自动调节。
2. **控制精度低**：仅提供档位操作，无法精确维持在目标湿度范围。
3. **缺乏异常告警**：无法及时发现设备故障或环境异常。
4. **无数据记录**：用户无法查看历史湿度变化趋势，难以优化使用习惯。

SHC 项目正是针对上述痛点，利用物联网技术打造一款低成本、高精度、可扩展的智能湿度控制器。

### 2.2 战略定位

SHC 定位为一款面向 **智能家居入门级市场** 的嵌入式控制设备，具备以下核心特征：

- **低成本**：选用成熟且性价比高的 ESP32 平台和国产传感器 AHT20，BOM 成本控制在合理范围。
- **易扩展**：基于模块化设计，为 V2.0（联网）和 V3.0（智能化）预留充足的升级空间。
- **用户友好**：通过 OLED 显示屏和 LED 指示灯提供清晰的状态反馈，让用户在任何场景下都能直观了解设备工作状态。
- **开环兼容**：V1.0 版本独立运行，无需依赖网络或手机 App，降低用户使用门槛。

### 2.3 商业模式

- **硬件销售**：主控板 + 加湿器组合套件，或单独的控制器模组供创客/Diy 爱好者使用。
- **增值服务**（V3.0 之后）：基于云端数据的 AI 分析报告、设备远程运维服务。
- **生态接入**：兼容 Home Assistant 等主流智能家居平台，增强产品互联互通能力。

## 3. 目标用户分析

### 3.1 用户画像

| 用户类型 | 特征描述 | 核心需求 | 备注 |
|----------|----------|----------|------|
| **家庭用户** | 有婴幼儿、老人或鼻炎/过敏患者的家庭 | 保持室内湿度稳定，设备操作简单，安全可靠 | V1.0 核心用户群 |
| **数码爱好者/创客** | 对智能家居感兴趣，有基本动手能力 | 开源可定制，支持二次开发，数据可导出 | 社区传播主力，关注 V2.0+ |
| **办公室/小型商户** | 有密闭办公空间、小型店铺的用户 | 多设备管理、远程监控、异常告警 | V2.0/V3.0 目标用户 |
| **植物爱好者** | 养护热带植物或温室植物的用户 | 环境参数的精细控制，温湿度联合调控 | 细分市场 |

### 3.2 用户使用场景

**场景一：家庭卧室夜间加湿**

用户睡前发现湿度为 38%，低于舒适区间。按下 Button A 切换至自动模式，SHC 检测到湿度低于 45% 阈值，自动开启加湿器，LED 变为蓝色指示"加湿中"。OLED 屏幕实时显示温湿度变化。凌晨湿度回升至 48%，设备自动关闭加湿器，LED 恢复绿色。用户整夜无需干预。

**场景二：传感器故障告警**

AHT20 传感器因灰尘覆盖导致通信失败。SHC 检测到 I2C 通信超时，立即触发蜂鸣器鸣响报警，同时 LED 切换为红色常亮，OLED 显示 "Sensor Error" 字样。用户按任意按钮可消除蜂鸣告警，并检查传感器连接状态。

**场景三：远程查看与控制（V2.0）**

用户在公司办公，想起家中加湿器可能未关闭。通过手机浏览器打开 SHC 的 Web 页面，查看当前湿度为 52%，确认在舒适范围，放心继续工作。如需提前关闭加湿器，也可在页面上一键操作。

**版本发布策略：**
- V1.0 为稳定基座版本，采用 **滚动冻结策略**，确保核心闭环控制无 bug 后，冻结基础代码再启动 V2.0。
- V2.0 重点完成网络功能，预留向下兼容接口。
- V3.0 在 V2.0 基础上增加智能分析能力，后台可做云端方案，也可做边缘计算方案。

## 5. 功能需求详述

### 5.1 V1.0 MVP 功能需求（Must Have）

#### FR-001 温湿度实时采集

| 属性 | 说明 |
|------|------|
| **需求ID** | FR-001 |
| **优先级** | P0 - Must Have |
| **描述** | 系统通过 I2C 总线读取 AHT20 传感器数据，获取当前环境的温度值和相对湿度值 |
| **采集周期** | 每 2 秒采集一次 |
| **温度范围** | 0°C ~ 50°C（室内环境范围） |
| **湿度范围** | 0%RH ~ 100%RH |
| **精度要求** | 温度 ±0.3°C，湿度 ±2%RH（AHT20 标称精度） |
| **异常处理** | 连续 5 次通信失败（10 秒无数据），判定为传感器异常，触发报警 |
| **验收标准** | 连续运行 72 小时，数据采集无中断，误差在标称范围内 |

#### FR-002 OLED 状态显示

| 属性 | 说明 |
|------|------|
| **需求ID** | FR-002 |
| **优先级** | P0 - Must Have |
| **描述** | 使用 0.96 寸 128x64 I2C OLED 显示屏，持续刷新显示环境温湿度及系统状态 |
| **刷新频率** | 每 2 秒刷新一次（与采集周期同步） |
| **显示内容** | 湿度值（Humidity: XX%）、温度值（Temperature: XX.X°C）、工作状态（Status: ON/OFF/AUTO）、异常信息 |
| **OLED 亮度** | 默认 50% 亮度，可通过长按 Button B 切换亮度档位 |
| **熄屏策略** | 设备正常运行时不熄屏；可通过特定按键序列进入省电熄屏模式 |
| **验收标准** | 显示内容与串口输出数据一致，字符清晰无闪烁 |

**显示布局参考：**

```
┌──────────────────┐
│ Smart Humidifier  │  ← 标题行
│ Humidity:  48%    │  ← 湿度数据
│ Temp:      26.3°C │  ← 温度数据
│ Status:    AUTO   │  ← 工作模式/状态
│ [  Green LED  ]   │  ← 状态指示（可用小图标）
└──────────────────┘
```

#### FR-003 自动加湿控制（滞回控制）

| 属性 | 说明 |
|------|------|
| **需求ID** | FR-003 |
| **优先级** | P0 - Must Have |
| **描述** | 根据当前湿度值与设定阈值，自动控制继电器通断，实现加湿器的智能开关 |
| **控制策略** | 滞回控制（Hysteresis Control） |
| **触发开启** | 湿度 < 45%RH 时，继电器闭合（加湿器通电开始工作） |
| **触发关闭** | 湿度 > 55%RH 时，继电器断开（加湿器断电停止工作） |
| **滞回区间** | 45%~55%RH，在此范围内维持当前状态不切换 |
| **继电器规格** | 单路继电器模块，触点容量 ≥ 10A/250VAC |
| **保护机制** | 继电器切换最小间隔 ≥ 30 秒，防止频繁通断损坏加湿器 |
| **手动模式** | 用户可通过按键切换到手动模式，独立控制继电器 ON/OFF |
| **验收标准** | 在测试箱中验证：设初始湿度 60% → 开启除湿模拟使湿度下降 → 44%RH 时继电器应闭合 → 关闭除湿自然回升 → 56%RH 时继电器应断开。循环测试 100 次无异常 |

**滞回控制曲线说明：**

```
继电器状态
   ON    ████████████████
                          ████████████████
   OFF                     ████████████████
        └────┴────┴────┴────┴────┴────┴────→ 湿度(%RH)
           40   45   50   55   60
                ↑         ↑
            开启阈值     关闭阈值
       (滞回区间: 45%~55%)
```

#### FR-004 LED 状态指示灯

| 属性 | 说明 |
|------|------|
| **需求ID** | FR-004 |
| **优先级** | P0 - Must Have |
| **描述** | 使用三色 LED（红/绿/蓝）指示设备当前运行状态 |
| **颜色定义** | • **绿色常亮**：湿度在正常范围（45%~55%），系统正常运行<br>• **蓝色常亮**：加湿器正在工作中（继电器吸合）<br>• **红色常亮**：传感器异常或 Wi-Fi 连接失败<br>• **红色闪烁（1Hz）**：湿度超出异常阈值（<30% 或 >80%）<br>• **三色交替闪烁**：系统初始化中 |
| **LED 驱动** | 通过 GPIO 引脚直接驱动，或使用限流电阻 + 三极管驱动 |
| **验收标准** | 各状态切换时 LED 颜色变化正确，响应时间 ≤ 500ms |

#### FR-005 蜂鸣器报警

| 属性 | 说明 |
|------|------|
| **需求ID** | FR-005 |
| **优先级** | P0 - Must Have |
| **描述** | 当系统检测到异常状态时，有源蜂鸣器发出告警音 |
| **触发条件** | ① 传感器通信故障（I2C 超时）<br>② Wi-Fi 连接失败（V2.0 起）<br>③ 湿度超出异常阈值（<30% 或 >80%）持续 5 分钟以上<br>④ 继电器异常（如连续切换超过 10 次/小时） |
| **报警模式** | 急促短鸣（200ms 间隔），持续鸣响 30 秒 |
| **消音操作** | 用户按下任意按钮可提前停止报警 |
| **报警锁定** | 根本原因未消除时，消音后每隔 5 分钟再次报警 |
| **验收标准** | 模拟各异常条件，蜂鸣器按预期触发并可通过按键消音 |

#### FR-006 按键交互

| 属性 | 说明 |
|------|------|
| **需求ID** | FR-006 |
| **优先级** | P0 - Must Have |
| **描述** | 通过两个物理按钮实现模式切换和恢复默认等基本交互 |
| **硬件要求** | 2 个轻触按键，带 10kΩ 上拉电阻，软件消抖（Debounce 时间 50ms） |
| **Button A 功能** | • **短按（<1s）**：切换工作模式（自动 → 手动开 → 手动关 → 自动循环）<br>• **长按（>3s）**：进入/退出 Wi-Fi 配置模式（V2.0 启用） |
| **Button B 功能** | • **短按（<1s）**：恢复湿度阈值到出厂默认值（45%~55%）<br>• **长按（>3s）**：清除所有设置参数，恢复出厂设置（需确认机制，再按一次确认） |
| **组合操作** | • **同时按下 A+B（>5s）**：系统软件重启 |
| **确认反馈** | 每次按键有效时，LED 闪烁一次作为反馈 |
| **验收标准** | 所有按键操作功能正常，无抖动误触发，长按/短按区分准确 |
