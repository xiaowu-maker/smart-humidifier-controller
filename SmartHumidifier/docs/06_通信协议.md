## 1. I2C 总线协议

### 1.1 总线拓扑

```
ESP32 (Master)
  │
  ├── GPIO21 (SDA) ──┬──────────────┐
  │                  │              │
  │                  ▼              ▼
  │              AHT20          SSD1306 OLED
  │              (0x38)         (0x3C)
  │
  └── GPIO22 (SCL) ──┬──────────────┘
                     │
                  [4.7kΩ 上拉电阻]
                     │
                    3.3V
```

### 1.2 设备地址

| 设备 | I2C 地址 | 备注 |
|------|----------|------|
| AHT20 | 0x38 (7-bit) | 出厂固定 |
| SSD1306 OLED | 0x3C (7-bit) | 常见默认地址，部分模块为0x3D |

### 1.3 总线参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 时钟频率 | 100 kHz (标准模式) | ESP32默认，稳定可靠 |
| 上拉电阻 | 4.7 kΩ | 大多数模块已内置 |
| 供电电压 | 3.3V | ESP32 I2C引脚为3.3V容忍 |
| 最大电容 | 400 pF | I2C标准，两个设备远低于此值 |

### 1.4 AHT20 通信时序

```
读取温湿度数据：

1. Master 发送 START
2. Master 发送 0x70 (地址0x38 << 1 | WRITE)
3. Master 发送 0xAC (触发测量命令)
4. Master 发送 0x33
5. Master 发送 0x00
6. Master 发送 STOP
7. 等待 80ms (测量时间)
8. Master 发送 START
9. Master 发送 0x71 (地址0x38 << 1 | READ)
10. Slave 返回 6 字节数据 + 1 字节 CRC
11. Master 发送 NACK + STOP

数据解析：
  byte[0] = 状态字
  byte[1] = 湿度高字节 [19:12]
  byte[2] = 湿度低字节 [11:4]
  byte[3] = 湿度/温度混合 [3:0 | 19:16]
  byte[4] = 温度高字节 [15:8]
  byte[5] = 温度低字节 [7:0]
  
  湿度 = ((byte[1] << 12) | (byte[2] << 4) | (byte[3] >> 4)) * 100 / 1048576
  温度 = (((byte[3] & 0x0F) << 16) | (byte[4] << 8) | byte[5]) * 200 / 1048576 - 50
```

### 1.5 OLED (SSD1306) 通信

```
数据发送格式：

控制字节 + 数据
│
├── 0x00 = 后续是命令字节
└── 0x40 = 后续是显示数据

初始化序列（简化）：
0xAE  Display OFF
0xD5  设置时钟分频
0xA8  设置多路复用比
0x8D  启用电荷泵
0xAF  Display ON
```

## 2. 串口通信协议

### 2.1 物理层

| 参数 | 值 |
|------|-----|
| 波特率 | 115200 bps |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验位 | 无 |
| 流控制 | 无 |

### 2.2 日志输出格式

```
[LEVEL] [TAG] [TIMESTAMP] message

示例：
[INFO]  [SENSOR]  [00:00:12] Temperature: 26.3°C, Humidity: 48.2%
[INFO]  [RELAY]   [00:00:12] Relay ON  (Humidity 44.8% < 45%)
[WARN]  [SENSOR]  [00:02:30] Sensor read failed, using cached value
[ERROR] [SENSOR]  [00:03:00] Sensor disconnected! 3 consecutive failures
[INFO]  [SYSTEM]  [00:00:00] Smart Humidifier Controller V1.0
[INFO]  [SYSTEM]  [00:00:01] I2C bus initialized
[INFO]  [SYSTEM]  [00:00:02] AHT20 sensor OK
[INFO]  [SYSTEM]  [00:00:03] OLED display OK
```

### 2.3 串口命令（V2.0+）

支持通过串口发送简单命令进行调试：

| 命令 | 格式 | 响应 | 说明 |
|------|------|------|------|
| STATUS | `STATUS` | JSON状态对象 | 获取当前状态 |
| RELAY ON | `RELAY ON` | `OK RELAY ON` | 手动开继电器 |
| RELAY OFF | `RELAY OFF` | `OK RELAY OFF` | 手动关继电器 |
| RESET | `RESET` | `Resetting...` | 软件复位 |
| THRESHOLD | `THRESHOLD 40 60` | `OK` | 修改阈值 |
| HELP | `HELP` | 命令列表 | 显示帮助 |

## 3. Web API 协议 (V2.0)

### 3.1 基础信息

| 项目 | 值 |
|------|-----|
| 协议 | HTTP/1.1 |
| 端口 | 80 |
| 数据格式 | JSON |
| 字符编码 | UTF-8 |

### 3.2 API 端点

#### GET /api/status — 获取系统状态

**请求**：
```
GET /api/status HTTP/1.1
Host: <esp32-ip>
```

**响应** (200 OK)：
```json
{
  "device": "SmartHumidifier",
  "version": "1.0.0",
  "timestamp": 1720694400,
  "uptime": 3600,
  "sensor": {
    "temperature": 26.3,
    "humidity": 48.2,
    "valid": true
  },
  "relay": {
    "on": false
  },
  "system": {
    "state": "RUNNING",
    "humidityLow": 45.0,
    "humidityHigh": 55.0
  },
  "network": {
    "rssi": -45,
    "ip": "192.168.1.100"
  }
}
```

**字段说明**：

| 字段 | 类型 | 单位 | 说明 |
|------|------|------|------|
| device | string | - | 设备名称 |
| version | string | - | 固件版本 |
| timestamp | number | Unix秒 | 数据采集时间 |
| uptime | number | 秒 | 系统运行时间 |
| sensor.temperature | number | ℃ | 温度 |
| sensor.humidity | number | %RH | 相对湿度 |
| sensor.valid | boolean | - | 数据是否有效 |
| relay.on | boolean | - | 继电器状态 |
| system.state | string | - | 系统状态：INIT/RUNNING/HEATING/SENSOR_ERROR |
| system.humidityLow | number | %RH | 开启阈值 |
| system.humidityHigh | number | %RH | 关闭阈值 |
| network.rssi | number | dBm | WiFi信号强度 |
| network.ip | string | - | 设备IP地址 |

---

#### GET /api/history — 获取历史数据（V2.0+）

**请求**：
```
GET /api/history?hours=24 HTTP/1.1
Host: <esp32-ip>
```

**响应** (200 OK)：
```json
{
  "points": 432,
  "interval": 120,
  "data": [
    {"t": 1720608000, "temp": 26.1, "hum": 47.5},
    {"t": 1720608120, "temp": 26.2, "hum": 47.3},
    ...
  ]
}
```

---

#### POST /api/control — 发送控制指令（V2.0+）

**请求**：
```
POST /api/control HTTP/1.1
Host: <esp32-ip>
Content-Type: application/json

{
  "action": "setThreshold",
  "humidityLow": 40.0,
  "humidityHigh": 60.0
}
```

**支持的操作**：

| action | 参数 | 说明 |
|--------|------|------|
| `setThreshold` | humidityLow, humidityHigh | 修改湿度阈值 |
| `relayOn` | - | 手动开启继电器 |
| `relayOff` | - | 手动关闭继电器 |
| `setAuto` | - | 恢复自动模式 |
| `restart` | - | 软件重启 |

**响应** (200 OK)：
```json
{
  "success": true,
  "message": "Threshold updated"
}
```

**错误响应** (400 Bad Request)：
```json
{
  "success": false,
  "error": "Invalid threshold values"
}
```

## 4. MQTT 协议 (V3.0)

> 本章为 V3.0 版本规划内容，V1.0 不实现。

### 4.1 连接参数

| 参数 | 值 |
|------|-----|
| Broker | 用户配置（默认 `homeassistant.local`） |
| 端口 | 1883 (TCP) / 8883 (TLS) |
| Client ID | `smart_humidifier_<MAC>` |
| Keep Alive | 60s |
| QoS | 1 (至少一次) |

### 4.2 Topic 设计

```
发布 (ESP32 → Broker)：

shc/<device_id>/temperature     {"value": 26.3, "unit": "celsius"}
shc/<device_id>/humidity        {"value": 48.2, "unit": "percent"}
shc/<device_id>/relay           {"state": "ON"}
shc/<device_id>/status          {"state": "RUNNING", "uptime": 3600}
shc/<device_id>/availability    "online" / "offline"  (LWT)

订阅 (Broker → ESP32)：

shc/<device_id>/command/relay   "ON" / "OFF" / "AUTO"
shc/<device_id>/command/threshold {"low": 40, "high": 60}
shc/<device_id>/command/restart ""
```

### 4.3 Home Assistant 自动发现

```
Topic: homeassistant/sensor/shc_<mac>/humidity/config
Payload:
{
  "device_class": "humidity",
  "name": "SHC Humidity",

  "unit_of_measurement": "%",
  "value_template": "{{ value_json.value }}",

  "device": {
    "identifiers": ["shc_<mac>"],
    "name": "Smart Humidifier Controller",

}
```

## 5. 数据格式规范

### 5.1 数值精度

| 数据类型 | 精度 | 示例 |
|----------|------|------|
| 温度 | 1位小数 | 26.3 |
| 湿度 | 1位小数 | 48.2 |
| 时间戳 | 整数 (Unix秒) | 1720694400 |
| 运行时间 | 整数 (秒) | 3600 |
| 百分比 | 1位小数 | 45.0 |

### 5.2 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| JSON字段 | camelCase | `humidityLow`, `humidityHigh` |
| Topic | snake_case | `shc/device_001/temperature` |
| 日志标签 | UPPER_SNAKE | `[SENSOR]`, `[RELAY]` |
| 状态枚举 | UPPER_SNAKE | `RUNNING`, `SENSOR_ERROR` |

### 5.3 时间戳规范

- 所有时间戳使用 **Unix 时间戳**（UTC秒）
- 设备本地时间通过 NTP 同步
- 无网络时使用设备启动后的相对时间
