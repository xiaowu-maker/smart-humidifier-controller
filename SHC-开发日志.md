#### 3. 固件开发（下午～晚上）

编写了 **7 个固件模块**，全部编译通过：

| # | 模块 | 文件 | 功能 | 亮点 |
|:--|:--|:--|:--|:--|
| 1 | **Logger** | `Logger.h/cpp` | 4级日志系统 | 宏封装 + 编译期可关闭 |
| 2 | **SerialCommand** | `SerialCommand.h/cpp` | 串口命令解析 | 注册制 + 非阻塞 + 大小写不敏感 |
| 3 | **SensorManager** | `SensorManager.h/cpp` | AHT20 传感器 | 数据验证 + 跳变过滤 + 校准偏移 |
| 4 | **DisplayManager** | `DisplayManager.h/cpp` | OLED 3页面 | 反白标题栏 + 页面指示器 + °符号处理 |
| 5 | **ActuatorManager** | `ActuatorManager.h/cpp` | 三合一执行器 | Relay(保护) + LED(非阻塞闪) + Buzzer(模式) |
| 6 | **ButtonManager** | `ButtonManager.h/cpp` | 双按钮检测 | 消抖 + 单击/双击/长按/超长按 |
| 7 | **main.cpp** | `main.cpp` | 主程序入口 | 滞回控制 + 手动/自动 + 7个命令 |

**技术栈**：
- `platformio.ini`：ESP32 esp32dev + Arduino Framework + 5个Adafruit库 + ArduinoJson
- 面向对象设计：每个硬件对应一个 C++ 类
- 非阻塞编程：全程 `millis()` 计时，无 `delay()` 阻塞
- 优雅降级：传感器故障不影响系统运行，使用缓存数据 + 告警

详见 [[SHC-代码模块索引]] 和 [[SHC-固件架构详解]]

### 🐛 遇到的 Bug 与修复

今天遇到了 **4 个编译错误**，全部修复。每个都是一个学习点：

| # | Bug | 根因 | 知识点 |
|:--|:--|:--|:--|
| 1 | `Wire was not declared` | 缺少 `#include <Wire.h>` | 每个 Arduino 库都有自己的头文件依赖 |
| 2 | 三元运算符类型不匹配 | `F("OK")` 返回 `__FlashStringHelper*`，与 `const char*` 不兼容 | C++ 类型系统严格，三元运算符要求两侧类型一致 |
| 3 | 未使用变量警告 | 声明了 `int s` 但从未使用 | 编译器会检查未使用变量，养成清理代码的习惯 |
| 4 | `LedColor::WHITE` 冲突 | SSD1306 库的 `#define WHITE` 宏把枚举值替换成了 `1` | **宏是全局的**，不尊重命名空间——这是经典的 C++ 预处理器陷阱 |

详见 [[SHC-编译与Bug记录]]

### 🎓 今日学习要点

1. **软件工程流程**：需求→架构→编码→测试→发布，不是"打开 IDE 就写"
2. **分层架构**：驱动层→HAL层→工具层→应用层，每层只做自己的事
3. **非阻塞编程**：用 `millis()` 代替 `delay()`，让 CPU 可以同时做多件事
4. **优雅降级**：硬件故障时系统不崩溃，尽量维持基本功能
5. **C++ 宏陷阱**：`#define` 不尊重命名空间，要小心命名冲突
6. **滞回控制**：用两个阈值（45%/55%）而非一个点（50%）来避免频繁开关
